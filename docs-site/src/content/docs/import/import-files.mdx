<Callout variant="whentouse">
	You have one or more files — PDFs, images, a Word doc, a scan — that need to become part of a workspace.
</Callout>

## Steps

1. Drag files onto the empty-state screen or an open workspace, **or**
2. Use **Add Files** (<Kbd>⇧⌘O</Kbd>) and select one or more files, **or**
3. Use **Add Folder to Workspace…** to bring in every supported file in a folder at once.

<Figure src="/Orifold/assets/screenshots/import-files-overview.png" alt="Files mid-drag over the empty-state screen: the drop zone is highlighted blue and reads 'Release to import files', with a demo PDF hovering above it." caption="Dropping files onto the empty-state screen — release to import." />

## Supported formats

PDF, Word documents, HTML, Markdown, plain text, CSV, JSON, XML, and common image formats (PNG, JPEG, and others).

<Callout variant="note">
	A workspace holds up to **50 files**. Importing a folder adds its supported files up to that limit. If you're consistently hitting it, split the job into two workspaces and export each separately.
</Callout>

When a folder import finishes, Orifold summarizes how many files it imported — and VoiceOver announces the same number. The count is the number that actually landed in the workspace, not the number it set out to try, so skipped or unsupported files are never counted in.

<Callout variant="tip">
	If a PDF is corrupted and the native reader gives up on it, Orifold tries a qpdf recovery pass automatically — files that Preview won't open are often worth trying here.
</Callout>

## What you should see

Every imported file appears as its own document in the sidebar, ready to reorder, edit, or merge with the others.
