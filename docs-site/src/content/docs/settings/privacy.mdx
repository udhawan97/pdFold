**The promise:** there is no server. Nothing about your files or your use of Orifold is sent anywhere.

Here's the proof.

## Everything runs on your Mac

Every feature — import, page organization, text editing, annotation, OCR, compression, encryption, signing, form filling, and export — executes entirely on-device. There is no cloud step to opt out of, because there isn't one to opt into.

## Sandboxed by design

Orifold runs inside the macOS App Sandbox with exactly three entitlements:

```
com.apple.security.app-sandbox
com.apple.security.files.user-selected.read-write
com.apple.security.files.bookmarks.app-scope
```

The file entitlements mean Orifold can only read and write files **you** explicitly select — it has no standing access to the rest of your disk. The app-scope bookmark lets the Recently Viewed shelf reopen a file you already granted access to, without asking again — the access you gave, remembered, nothing wider.

## Zero telemetry

There is no analytics pipeline in Orifold, and no server to send data to even if there were. Usage isn't measured, tracked, or reported.

## Verify it yourself

Orifold is open source. Read the code, watch network activity while using the app (you'll find none), and check the entitlements file directly in the repository.
