Bug reports and feature requests are welcome in [Issues](https://github.com/udhawan97/Orifold/issues).

## Onboarding is one command

If you can build from source, you're already onboarded — there's no separate contributor setup. See [Build from source](/Orifold/developers/build/): if `swift test` passes, you're set up.

## Before opening a pull request

- Run the full [release gate](/Orifold/developers/release-gate/) locally.
- Keep localization coverage intact across all six languages if you touch user-facing strings.
- Prefer small, focused pull requests.
