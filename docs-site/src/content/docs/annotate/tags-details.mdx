## Workspace tags

1. Select a document in the sidebar.
2. Open the Inspector and choose **Tags**.
3. Add a tag to categorize the document inside this workspace.

Tags belong to the Orifold workspace. They help with organization but are not written as standard PDF metadata.

## PDF document properties

1. Select the document whose properties you want to change.
2. Open **Inspector → Info**.
3. Edit **Title**, **Author**, **Subject**, or **Keywords**.
4. Click **Apply metadata**.

The change is undoable and is written into a normal PDF export. In a workspace holding several files, the Info tab names the file whose properties you're editing — always the first file, because a merged PDF can carry only one document-info dictionary and the first file's is the one a merged export keeps.

Typing you haven't applied yet is kept. Switch to another inspector tab, click a page belonging to a different file, or undo something, and your in-progress edits are still waiting in the fields when you come back.

<Callout variant="note">
	If the PDF also contains XMP metadata, Orifold warns that some readers may prefer that separate metadata packet. The editor updates the standard PDF document-info fields; it does not rewrite the XMP packet.
</Callout>

## What you should see

The tag stays with the workspace, while applied document properties appear in the exported PDF's metadata.
